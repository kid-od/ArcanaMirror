# Arcana Mirror

Arcana Mirror 是一个偏向自我观照的塔罗网页应用。它希望把“抽牌”做成一种缓慢、安静、有仪式感的体验，把“解读”做成一种温和、可反思、不过度宿命化的阅读过程。

这不是一个强调恐惧、绝对预言或戏剧化宿命的产品。  
它更像一个用卡牌帮助你整理问题、理解情绪、看见模式的数字空间。

## 网页功能导览

### 1. 首页

首页负责建立整个产品的氛围与入口。

你可以在这里：

- 了解 Arcana Mirror 的整体气质与定位
- 选择进入单张牌阅读或三张牌阵阅读
- 浏览卡牌库
- 阅读关于塔罗使用方式的说明

### 2. 单张牌阅读

适合一个当下很明确、很集中的问题。

这个页面支持：

- 输入你自己的问题
- 使用预设问题提示
- 进入一段更轻量的仪式流程
- 抽出一张牌，得到更聚焦的阅读

这类阅读更适合：

- 情绪 check-in
- 某个决定前的短暂停顿
- 想看清“此刻最需要被看见的是什么”

### 3. 三张牌阵阅读

适合需要上下文、变化过程和时间感的问题。

这个页面支持：

- 围绕过去 / 现在 / 未来进行抽牌
- 看见一个问题是如何形成、如何停留在当下、以及下一步可能如何展开
- 阅读整组牌之间的关系，而不只是单张牌的独立含义

这类阅读更适合：

- 一段关系的变化
- 工作或学习中的阶段性问题
- 某个重复出现的模式
- 想理解“我现在正处在哪个过程里”

### 4. 仪式流程页面

这是网站体验里最重要的一段。

用户会从“提问”逐步进入：

1. 让问题成形
2. 进入仪式感更强的抽牌区域
3. 洗牌 / 选择 / 封牌
4. 在当前页面继续向下进入揭示阶段

这部分不是单纯点击按钮出结果，而是希望把“抽牌”做成一个有节奏的过程。

### 5. 揭示阶段

揭示阶段不是完整结果页，而是“第一次看见这组牌”。

在这里会优先呈现：

- 牌面本身
- 正位 / 逆位
- 关键词
- 非常简洁的第一印象
- 一句反思提示

这一步的目的，是先让用户和卡牌相遇，而不是立刻把所有解释一次性讲完。

### 6. 完整阅读结果页

结果页是更深一层的阅读。

当前结果页的设计重点是“从精炼到复杂”的阅读顺序：

- 先看牌
- 再看这张牌为什么落在这里
- 再看这张牌正在说什么
- 最后进入更完整的整合阅读与反思提示

单张牌阅读和三张牌阵阅读在这里会有不同层次：

- 单张牌更强调聚焦、语境与展开
- 三张牌更强调位置关系、整体模式、张力与过渡

### 7. 卡牌库

卡牌库用于帮助用户在抽牌前后安静地理解牌义。

这里支持：

- 浏览全部卡牌
- 按大阿尔卡那或花色筛选
- 查看卡牌详情
- 在中英文之间切换内容

卡牌详情页会提供：

- 卡牌名
- 关键词
- 正位 / 逆位基础含义
- 情绪、关系、事业等维度的延伸理解

### 8. 关于页面

关于页会说明：

- 这个产品如何理解塔罗
- 为什么这里强调“象征性反思”，而不是“固定命运”
- 如何看待正位与逆位
- 使用产品时应保持怎样的心理预期

### 9. 中英文切换

网站支持中文与英文双语切换。

切换后会同步影响：

- 导航与页面文案
- 抽牌流程说明
- 卡牌名称与卡牌描述
- 阅读结果中的解读文本

## 推荐使用方式

如果你是第一次使用这个网站，可以按下面的顺序体验：

1. 在首页先选择适合自己的阅读方式
2. 输入一个足够诚实、足够具体的问题
3. 在仪式流程里慢一点，不着急进入结果
4. 先看揭示阶段给出的第一印象
5. 再打开完整阅读页，进入更深的解读
6. 如果某张牌一直留在心里，再回到卡牌库查看更完整的背景含义

## 这个网站更适合什么样的问题

比较适合：

- “我现在真正需要理解什么？”
- “这段关系此刻更深的课题是什么？”
- “我在工作或学习上最值得留意的是什么？”
- “我正被邀请去理解自己哪一部分？”

不适合期待它直接回答：

- 固定的命运结论
- 恐惧式预言
- 医疗、法律、财务等专业判断

## 当前网页包含的主要页面

- `/`：首页
- `/draw/single`：单张牌阅读
- `/draw/three`：三张牌阵阅读
- `/reading/[id]`：阅读结果页
- `/cards`：卡牌库
- `/cards/[slug]`：卡牌详情页
- `/about`：关于与说明

## 技术说明

这个仓库是一个 `pnpm workspace` monorepo：

- `apps/web`: Next.js 前端
- `apps/api`: NestJS 后端
- `PostgreSQL + Prisma`: 数据存储与 ORM

前端负责：

- 页面与交互体验
- 抽牌流程
- 结果页展示
- 国际化文案呈现

后端负责：

- 卡牌目录
- 阅读生成
- 阅读结果结构
- 数据持久化

## 本地开发

### 环境要求

- Node.js `22.x`
- pnpm `10.x`
- Docker

### 安装依赖

```bash
pnpm install
```

### 启动数据库

```bash
docker compose up -d
```

默认 PostgreSQL：

- host: `localhost`
- port: `5432`
- database: `myapp`
- user: `postgres`
- password: `postgres`

### 配置环境变量

前端：

复制 `apps/web/.env.example` 为 `apps/web/.env.local`

```bash
NEXT_PUBLIC_API_BASE_URL=http://localhost:3001
```

后端：

复制 `apps/api/.env.example` 为 `apps/api/.env`

```bash
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/myapp?schema=public
CORS_ORIGIN=http://localhost:3000
JWT_ACCESS_SECRET=replace-with-a-long-random-secret
```

### 执行数据库迁移

```bash
pnpm prisma:migrate:deploy:api
```

### 启动前后端

```bash
pnpm dev:web
pnpm dev:api
```

默认地址：

- Web: `http://localhost:3000`
- API: `http://localhost:3001`
- API Docs: `http://localhost:3001/docs`

## 常用命令

```bash
pnpm dev:web
pnpm dev:api

pnpm build:web
pnpm build:api

pnpm start:web
pnpm start:api

pnpm prisma:generate:api
pnpm prisma:migrate:deploy:api
```
